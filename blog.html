• 1. Discuss the scope of var, let, and const

    In JavaScript, the scope of variables declared with var, let, and const differs.

    • var

      Function Scope: Variables declared with var are function-scoped, meaning they are only accessible within the function in which they are declared. If declared outside a function, they become global variables, accessible from anywhere within the script.


      Hoisting: Variables declared with var are hoisted, which means they are moved to the top of their containing function or global scope during the compilation phase. However, their initial value is set to undefined. This can lead to unexpected behavior if you try to access the variable before it's declared.


    • let

      Block Scope: Variables declared with let are block-scoped, meaning they are limited to the block in which they are defined (blocks include if statements, loops, and functions). They are not accessible outside the block.


      No Hoisting: Unlike var, variables declared with let are not hoisted. If you try to access them before declaration, you'll get a ReferenceError.


    • const

      Block Scope: Variables declared with const also have block scope, just like let.


      No Reassignment: Variables declared with const cannot be reassigned after declaration. However, they can still be mutable if they hold reference types like objects or arrays. This means you can modify the properties or elements of a const object or array, but you cannot assign it a completely new value.


      No Hoisting: Similar to let, const variables are not hoisted, and accessing them before declaration results in a ReferenceError.


  • 2. Tell us the use cases of null and undefined

    Null

    Null is a special value in JavaScript used to represent the intentional absence of any object value. It has several use cases:

    • It can indicate that a variable or property has no meaningful value.
    • It can be used as a placeholder for an object or value that will be assigned later.
    • It can be set to explicitly clear or reset values.

    Undefined

    Undefined, on the other hand, is another distinct value in JavaScript with its own set of use cases:

    • It's often used to represent the absence of an argument in a function call, especially with default parameter values.
    • When trying to access an object property that doesn't exist, it returns undefined.
    • If a function doesn't have a return statement or returns nothing explicitly, it returns undefined.
    • Variables declared without initialization are automatically set to undefined.

  • 3. What do you mean by REST API?

    • A REST API, or Representational State Transfer Application Programming Interface, is a set of rules and conventions for building and interacting with web services. It is an architectural style that relies on the HTTP protocol for communication between clients and servers.

    Key characteristics of a REST API include:

    • Stateless: Each request from a client to a server must contain all the information needed to understand and process the request. The server does not store any client state between requests.
    • Resource-Based: Resources (e.g., data objects or services) are identified by URIs (Uniform Resource Identifiers) and can be operated on using standard HTTP methods such as GET, POST, PUT, and DELETE.
    • Representation: Resources can have multiple representations, such as JSON or XML, and clients can specify the desired representation in the request.
    • Stateless Communication: Each request from a client to a server must be self-contained, meaning it should include all the information needed for the server to fulfill the request. There should be no dependency on prior requests.
    • Client-Server Architecture: The client and server are separate entities that communicate over a network. This separation of concerns allows for more scalability and flexibility.

    REST APIs are widely used for building web applications, mobile apps, and other distributed systems, providing a simple and efficient way to interact with data and services over the internet.
